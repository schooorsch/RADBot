RADBot – Privacy Policy

Last updated: March 2026

1. Data Collection

RADBot does not collect, store or share any personal data. No user information is logged or retained beyond the current session. All in-memory data (play, volume, current station) is reset when the bot restarts.

2. Voice & Text Channels

RADBot only reads slash commands directed at it. It does not read, store or process any other messages or voice data.

3. Third Party Services

RADBot streams audio from publicly available radio stations and uses the Radio Browser API to search for stations. We are not responsible for the content or availability of third party streams. Radio Browser does not receive any personal user data from RADBot.

4. Server Data

RADBot temporarily stores the following data in memory for the duration of a session only:

  • Guild ID and voice channel ID (required for audio playback)
  • Currently playing station per server
  • Volume setting per server
  • Play history per server (last 10 stations, in-memory only)

None of this data is written to disk or shared with third parties.

5. Logging

RADBot logs technical events (errors, stream health checks, dead streams) to local log files on the host server for debugging purposes only. These logs do not contain any user-identifiable information.

6. Children's Privacy

RADBot does not knowingly collect any data from users under the age of 13. As RADBot collects no personal data at all, it is safe for use in any Discord server that complies with Discord's own Terms of Service.

7. Changes to This Policy

This privacy policy may be updated occasionally. The latest version is always available at this URL. Continued use of RADBot after changes constitutes acceptance of the updated policy.

8. Contact

For any privacy concerns, contact .schorsch on Discord.
